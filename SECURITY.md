# Security policy

网络急救箱会读取并修改 Windows 网络配置，因此安全问题优先级高于普通功能问题。

## 报告漏洞

请使用 GitHub 仓库的 **Security → Report a vulnerability** 私密提交，不要在公开 Issue 中披露可利用细节、用户网络配置或敏感日志。

报告最好包括：

- 受影响版本与 Windows 版本；
- 问题的可复现步骤和影响；
- 最小化、已脱敏的日志或概念验证；
- 你认为可行的修复方向（可选）。

## 支持版本

当前只为最新 GitHub Release 提供安全修复。发现问题后，请先确认最新版仍可复现。

## 安全设计摘要

- Tauri 窗口启用严格 CSP，渲染层不能直接访问外部网络；
- 未安装 Shell 插件，前端只能调用编译进 Rust 的固定命令；
- 修复动作 ID、还原范围、快照编号和更新地址均在后端再次校验；
- 修复操作只能从编译期白名单中选择，不能从界面提交任意 PowerShell；
- 需要修改系统配置的动作才触发 UAC，执行前强制创建网络快照；
- 应用启用单实例保护，避免两个窗口同时写快照或执行修复；
- npm 与 Cargo 均使用锁文件，CI 同时执行 pnpm audit 和 RustSec 审计。
