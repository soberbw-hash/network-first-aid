# Security policy

网络急救箱会读取并修改 Windows 网络配置，因此安全问题优先级高于普通功能问题。

## 报告漏洞

请使用 GitHub 仓库的 **Security → Report a vulnerability** 私密提交，不要在公开 Issue 中披露可利用细节、用户网络配置或敏感日志。

报告最好包括：

- 受影响版本与 Windows 版本；
- 问题的可复现步骤和影响；
- 最小化、已脱敏的日志或概念验证；
- 你认为可行的修复方向（可选）。

## 支持版本

当前只为最新 GitHub Release 提供安全修复。发现问题后，请先确认最新版仍可复现。

## 安全设计摘要

- Electron 渲染进程启用沙箱和上下文隔离，关闭 Node.js 集成；
- 主窗口拒绝弹窗、webview、导航和权限申请；
- preload 只暴露固定 IPC 能力，主进程再次验证请求来源；
- 修复操作只能从编译期白名单中选择，不能从界面提交任意 PowerShell；
- 依赖安装使用锁文件，CI 会阻止高危依赖公告进入主分支。
